About

I am largely functionality focused. I build things to learn how they work, especially backend and low-level systems.

Why I build

I like learning how systems work and the best way to do that, is to make them yourself. I have fun in creating whatever I want and however I want it.

What I care about

  • Learning from each project and problem
  • Good system design, that is easy to use
  • Working code that is easy to read
  • Functionality over fancy appearance
