About

I build systems that are functionality focused. I’m especially interested in backend engineering, low-level software, and projects where I can learn more about how stuff works.

Why I build

I like learning how systems work and the best way to do that, is to make them yourself from end-to-end. I have fun in creating whatever I want and however I want it. I’m comfortable moving from architecture and implementation to debugging and refining the details until the work is completed.

What I care about

  • Learning from each project and problem
  • Good system design, that is easy to use
  • Working code that is easy to read
  • Functionality over fancy appearance
